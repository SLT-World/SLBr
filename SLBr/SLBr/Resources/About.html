The new, clean, lightweight .NET Web browser

SLBr

SLBr is the best open source .NET Web Browser made with WPF (C# and XAML) there is! More lightweight and faster than Google Chrome, uses less memory compared to most major browsers. With a semi-flat modern UI design and a lot of useful features, for example: Tab Unloading, which frees up resources and memory.

More shorter description: SLBr is an open source web browser that display webpages and files. Browse the web with the tiniest memory usage possible!

With the use of the high-performance .NET wrapper around the Chromium Embedded Framework (CEF), CEFSharp, things are made more easy to draw and render webpages in high quality.

Based on the powerful Chromium platform

With the powerful Chromium browser engine, SLBr isn't required to use Internet Explorer, download Chromium Edge to use WebView2 or a heavy customized browser engine as the main intergrated browser engine, while also having access to Chromium's rich and useful features that can be developed into much useful features and get implemented.

System Requirements

  • VC++ 2015 Runtime is required
  • .NET Framework 4.8 and above is required
  • Must not be Intel i5 processors
  • Segoe MDL2 Assets is required to be installed on Windows 7 and below

Important Information

SLT only supports open source media codecs and proprietary codecs are not supported. To build SLBr with proprietary codecs, you must first build CEF with proprietary codecs.

SLBr may not have any tutorials on the topic of building proprietary codecs. But here are some useful links that might help you with the building process:

  • CEF Master Build Quick Start Documentation
  • Build CEF with proprietary codecs support
  • CefSharp Docker
  • How to enable H264 support in CEF3 latest version
  • CEF Automated Build Setup Documentation

Features that keeps SLBr from being a memory hog it once was (Yes, SLBr used to be a memory hog just like Google Chrome)

  • A browser process of SLBr only starts when it's viewed
  • A Tab Unloading feature that unloads inactive but initialized browser processes
  • Limitations on the number of rendering processes there can be
  • Unnoticable and memory-hungry features of the Chromium browser engine are disabled for default
  • No proxy servers, no proxy resolving

Keyboard Shortcuts

  • Ctrl+R, F5 (Refresh)
  • Ctrl+T (New Tab)
  • Ctrl+W (Close Tab)
  • Ctrl+S (Screenshot)
  • F12 (Developer Tools)
