UI Changes

  • SLBr in Internet Explorer mode will no longer show urls in the tabs on startup
  • Removed the add button from the tabs header. To create a new tab, find and press the + icon in the tools bar
  • A more sharper text than ever before
  • A bit of border position change in the tools bar

Bug fixes

  • Reduced some crashes in the SLBr Internet Explorer mode
  • Fixed the homepage setting not saving
  • Switched to Software Rendering to eliminate the problem of SLBr not working on certain devices
  • SLBr can now open files through the file explorer (It is actually implemented, just that it used to cause SLBr to crash)
  • Fixed useragent not applying the SLBr useragent
  • Fixed new release prompt showing the current version and not the new available version
  • Values are now passed into weblight loading
  • Fixed a few no internet connection crashes

Code Changes

  • Changed the way of registering schemes, using classes for a more cleaner look, but sacrificed faster startup time
  • Improved the code
  • Removed unnecessary usings
  • Removed and changed the way on how the events of close & pin buttons are added. Slightly improved a bit of load time
  • Changed from initializing a instance of Uri for host to a fast and straightforward url host parsing
  • The maximum number of render processes is set from 1 to 5, to minimize the affects of render process crashes
  • Limited the size of disk cache to 100 MB
